How tamper detection works

1
All certificate fields are combined into one string in a fixed, sorted order.
2
SHA-256 hashes that string into 32 bytes. This is a one-way, avalanche function — changing even one character completely scrambles all 32 output bytes.
3
Those 32 bytes drive every visual property of the seal: background colour, petal count, spoke count, inner star shape, outer dot ring pattern. Same fields → always the same seal.
4
Anyone can regenerate the seal here and compare visually to the one on the certificate. The Certificate ID is the first 16 hex digits of the hash — a quick numeric check.
5
To forge a certificate: you'd have to find different field values that produce the exact same 256-bit hash — a task computationally impossible with SHA-256 (2²⁵⁶ possible outputs).
Try it: Fill in the original fields and verify. Then change the grade from "A+" to "B-" and verify again. The seal will look completely different — different colours, different shape count, different dot pattern. This is what an institution or employer sees when they check a tampered certificate.

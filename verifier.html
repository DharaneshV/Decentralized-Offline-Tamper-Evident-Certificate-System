How ECDSA Tamper-Evidence Works

1
Canonical Fields Bound: All fields are joined in a fixed alphabetical order with pipe delimiters (no JSON). This exact format is implemented identically in Python (signing) and JavaScript (verifying).
2
ECDSA P-256 Signature: The canonical string is hashed with SHA-256 and signed with the institution's private key. The 64-byte raw signature (r||s) is embedded in the PDF metadata.
3
Offline Verification: This page uses crypto.subtle.verify() with the embedded public key to check the signature — fully offline, no server, no network. Cryptographically impossible to forge without the private key.
4
Visual Deterrents: The geometric seal, security texture, and field tints are derived from the signature bytes. They're a quick-glance check and deterrent — the ECDSA signature is the actual cryptographic guarantee.
